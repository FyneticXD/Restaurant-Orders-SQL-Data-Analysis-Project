# Restaurant Orders Analysis - SQL

## 📊 Deskripsi Proyek

Proyek ini merupakan portofolio analisis data menggunakan **SQL (MySQL)** dengan memanfaatkan dataset transaksi restoran. Analisis dilakukan untuk mengeksplorasi performa menu, pola pemesanan pelanggan, dan tren penjualan sehingga menghasilkan insight yang dapat mendukung pengambilan keputusan bisnis berbasis data.

## 📂 Dataset

Dataset yang digunakan terdiri dari data menu dan transaksi pesanan restoran, meliputi:

- Menu Makanan dan Minuman
- Kategori Menu
- Harga Menu
- Detail Pesanan
- Tanggal Pesanan
- Jumlah Pesanan

## 💡 Analisis & Insight Utama

- **Eksplorasi Database**: Memahami struktur tabel dan data yang tersedia.
- **Analisis Menu**: Mengidentifikasi menu dengan harga tertinggi dan terendah, jumlah menu, serta distribusi menu berdasarkan kategori.
- **Analisis Pesanan**: Menganalisis jumlah pesanan, frekuensi pemesanan, dan pola pembelian pelanggan.
- **Analisis Penjualan**: Menghitung total penjualan serta mengidentifikasi menu yang paling banyak dan paling sedikit dipesan.
- **Analisis Kategori**: Membandingkan performa setiap kategori menu berdasarkan jumlah pesanan.
- **Business Insight**: Menghasilkan insight sebagai dasar evaluasi produk dan pengambilan keputusan bisnis.

## 🛠️ Teknologi yang Digunakan

- **Database**: MySQL
- **Bahasa Query**: SQL
- **Konsep SQL**:
  - SELECT
  - WHERE
  - ORDER BY
  - GROUP BY
  - JOIN
  - Aggregate Function (COUNT, SUM, AVG, MIN, MAX)
  - Subquery

## 🚀 Cara Menjalankan Proyek

1. Clone atau unduh repositori ini.
2. Import file `create_restaurant_db.sql` ke MySQL.
3. Jalankan seluruh file SQL secara berurutan.
4. Tinjau hasil query untuk memperoleh insight bisnis.

## 📈 Nilai Bisnis

Analisis ini membantu menjawab beberapa pertanyaan bisnis, seperti:

- Kategori menu mana yang memiliki performa terbaik?
- Menu apa yang paling sering dan paling jarang dipesan?
- Bagaimana pola pemesanan pelanggan?
- Produk mana yang perlu diprioritaskan untuk strategi pemasaran atau evaluasi menu?
